## AEO (Answer Engine Optimization) Strategies

### For Google AI Overviews
1. **Featured Snippet Format** - Structure content as direct answers in 40-60 word paragraphs
2. **Comparison Tables** - CEREC vs traditional crowns, Invisalign vs braces (tables get pulled into AI Overviews)
3. **Cost Ranges** - Include specific price ranges ("Invisalign typically costs $3,000-$7,000 in the Orlando area")
4. **Process Steps** - Number treatment steps (AI Overviews love numbered lists)
5. **FAQ Schema** - Every page should have 5-10 FAQs with schema markup
6. **E-E-A-T Signals** - Author bios with credentials on all content, link to doctor profiles

### For Perplexity AI
1. **Cite-Worthy Content** - Write content that Perplexity would want to cite as a source
2. **Factual Specificity** - Include specific data points, statistics, and credentials
3. **Unique Local Data** - "Only CEREC-equipped practice in Ocoee" type claims (if true)
4. **Structured Q&A** - Match the exact question format people ask Perplexity
5. **Authority Signals** - Mention certifications, memberships (ADA, FDA, Invisalign provider tier)
6. **Fresh Content** - Perplexity favors recently updated content; add a "Last Updated" date

### For ChatGPT (SearchGPT / Browsing)
1. **Conversational Content** - Write in natural Q&A format that matches how people prompt ChatGPT
2. **Complete Answers** - Provide comprehensive answers that don't require follow-up searches
3. **Local Context** - Include neighborhood context ("located near the intersection of Maguire Rd and Silver Star Rd in Ocoee")
4. **Comparison Content** - "Best dentist in Ocoee" type comparison content with differentiators
5. **Semantic Richness** - Use related terms and synonyms throughout (dental office, dental practice, dental clinic)

### For Voice Search (Alexa, Siri, Google Assistant)
1. **Speakable Schema** - Mark key content sections as speakable
2. **Question-Answer Pairs** - Target "Hey Google, find me a dentist in Ocoee" type queries
3. **Concise Answers** - First sentence of each FAQ should be a complete, speakable answer
4. **Phone Number Prominence** - Ensure phone number is easily discoverable for "call" voice commands
5. **Hours Clarity** - "Open today" queries depend on correct structured data

## Technical SEO Checklist

### Site Performance
- [ ] Page load time under 3 seconds on 3G
- [ ] Largest Contentful Paint (LCP) under 2.5 seconds
- [ ] First Input Delay (FID) / Interaction to Next Paint (INP) under 200ms
- [ ] Cumulative Layout Shift (CLS) under 0.1
- [ ] Lighthouse Performance score above 90
- [ ] Enable gzip/brotli compression
- [ ] Implement browser caching headers
- [ ] Minimize render-blocking CSS/JS
- [ ] Optimize images (WebP/AVIF, responsive srcset, lazy loading)
- [ ] Inline critical CSS, defer non-critical

### Crawlability & Indexing
- [ ] Submit XML sitemap to Google Search Console and Bing Webmaster Tools
- [ ] robots.txt allows crawling of all important pages
- [ ] Canonical URLs set on all pages
- [ ] No orphan pages (every page linked from at least one other page)
- [ ] Clean URL structure (no query parameters for main pages)
- [ ] 301 redirects for any old URLs
- [ ] No broken links (404s)
- [ ] Proper pagination if applicable

### Mobile & Accessibility
- [ ] Mobile-friendly test passes
- [ ] Tap targets at least 48x48px
- [ ] Font size minimum 16px on mobile
- [ ] No horizontal scrolling on any viewport
- [ ] WCAG 2.1 AA contrast ratios met
- [ ] All images have descriptive alt text
- [ ] Proper heading hierarchy (single H1, logical H2/H3 nesting)
- [ ] Skip navigation link for keyboard users
- [ ] ARIA labels on interactive elements

### Security & Trust
- [ ] HTTPS everywhere (SSL certificate active)
- [ ] HSTS header enabled
- [ ] Privacy policy page
- [ ] HIPAA compliance notice
- [ ] ADA accessibility statement

### Structured Data
- [ ] JSON-LD preferred over microdata
- [ ] No errors in Google Rich Results Test
- [ ] All schema types validated at schema.org validator
- [ ] Breadcrumb schema on all pages
- [ ] FAQ schema on relevant pages
